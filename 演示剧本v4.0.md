# 瀑布流教学演示系统 - 演示剧本 v4.0

## 核心理解修正

**正/负问题空间** = **在对话流中展开的特殊消息卡片**

就像我们的对话一样：
- 用户发送消息
- AI 在对话流中回复（可能是普通消息，也可能是正/负问题空间卡片）
- 正/负问题空间是**消息的一种特殊展示形式**，不是独立模块

## 核心概念：对话流中的消息类型

```
💬 AI 教学交互空间 = 连续的消息流（类似 ChatGPT）

消息类型：
├─ 🤖 system/assistant：普通教学消息
├─ 👤 user：学生输入
├─ 🎯 positive-question：正问题空间（绿色边框卡片）
└─ 💭 negative-question：负问题空间（红色边框卡片）

正/负问题空间是消息的特殊渲染形式，嵌入在对话流中
```

## 演示剧本（5幕）

### 第一幕：初始状态

**对话流**：
```

       本系统通过摄像头采集图像，经ESP32暂存后上传云端进行AI推理，
       最后在显示终端展示识别结果。

💡 提示：请在下方Scanvas中观察系统组件，准备开始连接数据流。
```

**Scanvas**：4个零散节点（摄像头、ESP32、云端、显示），无连线

| 场景 | 触发条件 | 对话流响应 | Scanvas | Mcanvas |
|------|----------|-----------|---------|---------|
| 正问题 | 用户输入含"为什么/什么是" | 🎯 绿色卡片展开 | - | - |

| 正确连线 | 学生修正连线 | ✅ 成功消息 | 连线变绿+闪烁 | 添加完成节点 |

## 组件职责（修正版）

| 组件 | 职责 |
|------|------|
| **DialogArea** | 对话流容器，支持多种消息类型渲染（普通/正问题/负问题） |
| **Message** | 单条消息组件，根据type渲染不同样式 |
| **PositiveQuestionCard** | 正问题空间卡片（绿色边框，可折叠） |
| **NegativeQuestionCard** | 负问题空间卡片（红色边框，可折叠，含代码） |
| **Scanvas** | 流程图画布，4个IoT节点，支持连线验证 |
| **Mcanvas** | 知识图谱，显示章节完成进度 |
| **Layout** | 三栏布局容器 |

## 数据流（修正版）

```
用户输入 → useDemo.sendMessage → 检测关键词/连线状态
                              ↓
                    ┌─────────┴─────────┐
                    ↓                   ↓
              正问题关键词          负问题触发
                    ↓                   ↓
         添加positive-question消息   添加negative-question消息
                    ↓                   ↓
              DialogArea渲染          DialogArea渲染
              🎯 绿色卡片             💭 红色卡片
```

## 消息数据结构

```typescript
interface Message {
  role: 'system' | 'user' | 'assistant';
  content: string;
  type?: 'normal' | 'positive-question' | 'negative-question';
  questionData?: {
    title: string;        // 问题标题
    content: string;      // 详细内容
    codeSnippet?: string; // 代码示例（负问题）
    hint?: string;        // 引导提示
    isCollapsed?: boolean; // 是否折叠
  };
}
```

## 与 v3.0 的关键区别

| 项目 | v3.0（错误） | v4.0（正确） |
|------|-------------|-------------|
| 正/负问题位置 | 独立模块在对话框下方 | 嵌入在对话流中作为消息 |
| 消息关系 | 正问题→AI回复→用户输入 | 用户输入→正问题卡片（AI回复的特殊形式） |
| 折叠机制 | 独立折叠按钮 | 卡片内嵌折叠，自动折叠 |
| 代码示例 | 独立代码块 | 嵌入在负问题卡片内 |

**核心理解**：正/负问题空间是**AI回复消息**的一种特殊展示形式，不是独立的功能模块。
